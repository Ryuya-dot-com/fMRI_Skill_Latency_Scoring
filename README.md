## 概要

### 実験の概要

参加者は動作を描いた絵を見て、架空言語「スプラング語」の進行形で動作を説明します。

- **12種類の動詞** × **25回反復** = **300試行/参加者**
- **5セッション** × **60試行/セッション**
- **21名の参加者**（合計 6,300 試行）
- 各録音 = 4秒の絵提示 + 3〜8秒のジッター（注視点）

### スプラング語の動詞一覧

| 動詞 | 日本語 | 進行形（正解） |
|------|--------|----------------|
| avanzar | 電話する | avanziando |
| comenzas | 笑う | camenziando |
| definis | 泣く | dofiniondo |
| deprimir | 走る | deprimiondo |
| dominas | 食べる | damitiando |
| gustar | 跳ねる | gustiando |
| impedir | 歌う | impediondo |
| lavar | 寝る | laviando |
| montas | 咳をする | mantiando |
| partir | 掃除をする | partiondo |
| recibis | 踊る | rocibiondo |
| sentis | 読む | sontiondo |

### 採点の目的

1. **Voice Onset Timing** — 録音中の発話開始点（ms）を正確に特定する
2. **Voice Offset Timing** — 発話終了点（ms）を記録し、発話持続時間を算出可能にする
3. **正確性** — 正しいスプラング語の進行形が産出されたかを判定する（1 / 0 / NR）
4. **Onset 欠損 trial の分類** — 発話が検出されない trial を「無発話」「機器不良」「非語彙音」の3カテゴリに再ラベルする

## データ準備

### 前提条件

- **Node.js** (v14+)
- **ffmpeg**（音声変換用）

### ビルドスクリプトの実行

```bash
cd Behavioral_Data/Scoring

# CSVデータ + 刺激画像のみ（音声変換なし）
node build/prepare-fmri-data.js

# 音声も WAV → MP3 に変換（初回のみ、約10分）
node build/prepare-fmri-data.js --with-audio
```

### ビルドスクリプトが行うこと

1. `Data_Audio/Results_XXXX/` の `stimulus_plan.tsv` から全300試行のメタデータを生成
2. `Analysis_Latency/analysis_events_table.csv` を読み込み、fMRI タイミング・活用規則・録音長などの全列を紐付け
3. 各参加者の CSV ファイルを `data/csv/{pid}/results_{pid}.csv` に出力（全23列）
4. WAV ファイルの左チャンネル（参加者マイク）を抽出し、MP3 に変換（64kbps, 22050Hz）
5. 刺激画像を `data/images/` にコピー
6. `data/participants.json`（インデックスファイル）を生成

### 生成されるデータ

```
data/
├── participants.json          # インデックス（参加者リスト、動詞情報）
├── csv/{pid}/results_{pid}.csv  # 各参加者の試行メタデータ
├── audio/{pid}/*.mp3            # 音声ファイル（300個/参加者）
└── images/*.jpg                 # 刺激画像（12枚）
```

## セットアップ画面

### 1. Rater ID を入力

自分の識別名を入力します（例: `rater_A`）。エクスポートファイルにこの名前が記録されます。

### 2. 参加者を選択

チェックボックスで採点する参加者を選びます（21名から任意に選択可能）。

- **Select All**: 全参加者を選択
- **Deselect All**: 全選択を解除

### 3. フィルター（任意）

セッションや動詞で採点範囲を絞り込めます。

| フィルター | 説明 |
|-----------|------|
| **Session** | Session 1〜5 から選択（例: Session 1 のみ → 60試行/参加者） |
| **Verb** | 特定の動詞のみ（例: avanzar のみ → 25試行/参加者） |
| **組み合わせ** | Session 1 + avanzar → 5試行/参加者 |

フィルターは採点画面のヘッダーからも変更可能です。

### 4. 採点を開始

**Start Scoring** ボタンを押すと、採点画面に進みます。

### 前回の続きから再開する場合

同じ Rater ID を入力すると、**Previous Session Found** と表示されます。**Resume** ボタンを押すと、前回の続きから再開できます。

## 採点の手順

各試行で以下の手順を繰り返します。

### ステップ 1: 音声を再生する

`Space` キーまたは **Play** ボタンで音声を再生します。

- 再生速度を変更したい場合は、速度ドロップダウン（0.5x〜1.5x）を使います
- `R` キーで onset 位置の 200ms 前から再生できます

### ステップ 2: 発話開始点（onset）を確認する

波形上の赤いマーカーが発話開始点を示しています。

| ボタン | キー | いつ使うか |
|--------|------|------------|
| **Confirm** | `C` | マーカー位置が正しい場合 |
| **Correct (drag)** | — | 赤い線をドラッグして位置を修正する場合 |
| **Manual Set** | — | 波形をクリックして手動で設定する場合 |
| **無発話** | `T` | 録音中に一切の発話がない場合 |
| **機器不良** | `F` | 機材トラブルにより録音不良の場合 |
| **非語彙音** | `G` | 咳・息・舌打ちなど非語彙音のみの場合 |

> **参考**: 緑のマーカー（4秒地点）は絵提示の終了位置です。発話は通常この付近で始まります。
> **ヒント**: NR を選択すると、onset は自動的に「無発話」に設定されます。既に他の No Speech カテゴリが選択されている場合はそちらが維持されます。

### ステップ 3: 正確性を採点する

- **1**（`1`キー）: 正しいスプラング語の進行形が産出されている
- **0**（`0`キー）: 不正確、途中で止めた、異なる単語を産出した
- **NR**（`9`キー）: 発話が全くない（onset 欠損 trial の3カテゴリから理由を選択）

### ステップ 4: メモを入力する（任意）

`N` キーでノート欄にフォーカスし、メモを入力できます。

### ステップ 5: 次の試行へ進む

`→` キーまたは `Enter` キーで次の試行に進みます。

### 1ステップ採点（高速モード）

`Shift` を押しながらスコアキーを押すと、**採点 + 自動で次の試行に移動** します。

| キー | 操作 |
|------|------|
| `Shift+1` | スコア 1（正確）+ 次の試行 |
| `Shift+0` | スコア 0（不正確）+ 次の試行 |
| `Shift+9` | スコア NR（無回答）+ 次の試行 |

> **効率的な採点フロー**: `Space`（再生）→ `C`（onset 確認）→ `Shift+1`（採点 + 次へ）の繰り返しで、最速で採点できます。

## 採点基準

### 1 — 正確

正しいスプラング語の進行形が産出されている場合。正解は画面の score hint に表示されます。

### 0 — 不正確

- 間違った活用形を産出した場合
- 途中で言い直した・止めた場合
- 全く異なる単語を産出した場合

### NR — 無回答（No Response）

録音に参加者の発話が全く含まれていない場合。以下の3カテゴリから理由を選択します：

- **無発話** — 参加者が一切発話しなかった
- **機器不良** — 機材トラブルにより録音不良
- **非語彙音** — 咳・息・舌打ちなど非語彙音のみ

## 発話開始点（onset）の確認

### onset とは

onset は、参加者が発話を開始した瞬間（ミリ秒単位）です。録音開始からの相対時刻で記録されます。

### 録音の構造

各録音は以下の構造を持ちます：

```
[0s]──── 絵提示（4秒） ────[4s]──── ジッター/注視点（3〜8秒） ────[7〜12s]
                               ↑ 緑マーカー
           ↑ 発話開始（通常ここの付近）
```

- **0〜4秒**: 絵が提示されている期間。参加者はこの間に発話を開始することが多い
- **4秒〜**: ジッター期間（注視点）。まだ発話していない場合はここで発話することもある
- **緑のマーカー**: 4秒地点を示す参照マーカー

### よくあるずれのパターン

| パターン | 原因 | 対処法 |
|----------|------|--------|
| **onset が早すぎる** | 環境ノイズやスキャナーノイズ | マーカーをドラッグして修正 |
| **onset が遅すぎる** | 小声での発話が見逃された | ズームインして確認、Manual Set |
| **息や舌打ちの誤検出** | 発話前の呼吸音やクリック音 | 実際の発話開始点にマーカーを移動 |
| **録音冒頭のノイズ** | MRI スキャナーのノイズ | 発話部分を見分けてマーカーを設定 |

## 発話終了点（offset）の確認

### offset とは

offset は、参加者の発話が終了した瞬間（ミリ秒単位）です。onset との差分が**発話持続時間**となります。

### 自動検出（補助）

音声ファイルの末尾から逆方向に走査し、RMS（二乗平均平方根）が閾値（0.01）を超える最後のサンプル位置を自動検出します（10ms ウィンドウ）。この値は**初期値の補助**として波形上に青いマーカーを配置しますが、fMRI 解析に使用すべき ground truth は、採点者が確認・修正した `offset_ms_rater`（エクスポート値）です。

- **青いマーカー**: 波形上に自動配置される offset マーカー（ドラッグで修正可能）
- **Auto-detected offset**: 採点パネルに自動検出値が参考表示される

> **重要**: 自動検出は末尾 RMS 探索に基づく近似値です。特に fMRI スキャナーノイズが録音末尾まで残る場合、offset が録音全長付近に検出されることがあります。**必ず採点者が目視で確認・修正してください。**

> **表示ルール**: 赤い onset が U1 です。発話が2つある場合は U2（オレンジ）のみ、3つの場合は U2（オレンジ）と U3（紫）を表示します。

### 修正方法

| 方法 | 説明 |
|------|------|
| **ドラッグ** | 青いマーカーをドラッグして位置を修正 |
| **手動入力** | Offset (ms) 入力欄に値を入力し **Apply** ボタンを押す |
| **確認** | Offset が正しい場合は **Confirm Offset** または `V` キーで確認 |

### duration QA

Speech duration は `Offset - Onset` で計算されます。250ms 未満または 3000ms 超の場合は、採点を止めずに `speech_duration_short` / `speech_duration_long` として QA に出力されます。

### データの信頼性階層

| レベル | データ | 用途 |
|--------|--------|------|
| **Ground truth** | `offset_ms_rater`（エクスポート値） | fMRI 解析・論文報告に使用 |
| **補助** | Auto-detected offset（UI 表示のみ） | 採点者の初期値として参考にする |

## Onset 欠損 trial の分類

発話が検出されない trial（onset が設定できない trial）は、以下の3カテゴリに分類します。

| カテゴリ | ボタン | キー | `onset_status` の値 | 説明 |
|----------|--------|------|---------------------|------|
| **無発話** | 無発話 | `T` | `no_speech_true` | 参加者が一切発話しなかった |
| **機器不良** | 機器不良 | `F` | `no_speech_technical` | 機材トラブルにより録音不良（マイク不具合、録音開始の失敗など） |
| **非語彙音** | 非語彙音 | `G` | `no_speech_nonlexical` | 咳・息・舌打ちなどの非語彙音のみで、発話として認められない |

### NR との関係

- **NR**（No Response）を選択すると、onset ステータスは自動的に「**無発話**」に設定されます
- 既に他のカテゴリ（機器不良 / 非語彙音）が選択されている場合、NR を押してもそちらが維持されます
- 3カテゴリのいずれかが選択された trial では、`onset_ms_rater` と `offset_ms_rater` は空欄になります

## データのエクスポート

### 参加者ごとの Excel 出力

画面下部の **Export Participant (.xlsx)** ボタンで、現在表示中の参加者のデータを Excel ファイルとしてダウンロードします。全試行を採点し終えると、自動的にダウンロードのポップアップが表示されます。

### 全参加者の一括 CSV 出力

**Export All (CSV)** ボタンで、選択した全参加者のデータを 1 つの CSV ファイルにまとめてダウンロードします。未採点の試行も含まれます（スコア列は空欄）。UTF-8 BOM 付きで出力されるため、Excel で直接開いても文字化けしません。

### fMRI events CSV 出力

**Export Events (CSV)** ボタンで、`pre_speech` / `speech` / `post_speech` / `utterance_start` を long format で出力します。

### QA CSV 出力

**Export QA (CSV)** ボタンで、解析前に確認すべき試行だけを抽出します。未採点、missing onset/offset、未確認 offset、250ms 未満または 3000ms 超の speech duration、DA コード不足、複数発話の marker 不足などが `qa_flags` に出力されます。

### JSON 出力

**Export JSON** ボタンで、全ての採点データを JSON 形式でダウンロードします。バックアップや後からの復元に利用できます。

#### 採点結果（ground truth — fMRI 解析にはこれらの値を使用）

| 列名 | 説明 |
|------|------|
| `accuracy_score` | 採点スコア（1, 0, または NR） |

| `first_speech_ms_rater` | 最初の発声開始点（ms）。`onset_ms_rater` と同じ値 |
| `utterance_onsets_ms` | U1 から順に並べた発話開始点（ms）。U1 は `onset_ms_rater`、U2 以降は追加U線 |

## よくある質問

### Q: 前回の続きから始めるには？

同じ Rater ID を入力すると、**Previous Session Found** が表示されます。**Resume** ボタンを押してください。

### Q: 採点を間違えた場合は？

該当の試行に戻って（`←` キーで前の試行へ）、スコアを上書きしてください。最新の値が自動的に保存されます。

### Q: 特定のセッションだけ採点したい

セットアップ画面のフィルターで Session を選択するか、採点画面のヘッダーにある「All S」ドロップダウンから Session を選択してください。

### Q: 特定の動詞だけ採点したい

セットアップ画面のフィルターで Verb を選択するか、採点画面のヘッダーにある「All V」ドロップダウンから動詞を選択してください。

### Q: 音声が再生されない場合は？

- ブラウザの音量がミュートになっていないか確認してください
- Google Chrome の使用を推奨します
- ページをリロードしてみてください
- 音声ファイルが `data/audio/` に存在するか確認してください

### Q: データが消えてしまった場合は？

- ブラウザの localStorage をクリアすると、採点データが削除されます
- シークレットモードではデータが保存されません
- 定期的に **Export JSON** でバックアップを取ることを推奨します

### Q: 色付きの線は何？

| 色 | 意味 |
|------|------|
| **赤** | 発話開始点（onset / U1）マーカー。ドラッグで修正可能 |
| **オレンジ** | U2 の追加発話開始点 |
| **紫** | U3 の追加発話開始点 |
| **緑** | U4 の追加発話開始点 |
| **青** | 発話終了点（offset）マーカー。自動検出され、ドラッグで修正可能 |
| **薄いグレー** | 絵提示の終了地点（4秒）。発話は通常この付近で始まる |

### Q: 「説明」ボタンはどこ？

ヘッダー左側の **説明** ボタン、または `I` キーで採点基準の詳細を確認できます。スプラング語の全動詞一覧も掲載されています。
